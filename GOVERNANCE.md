# OverlayKit Development Constitution

OverlayKit uses deterministic governance. The normative law is not this prose file; it is the
compiled contract under [`.overlaykit/governance`](.overlaykit/governance/README.md).

The contract consists of:

1. Immutable accepted decisions.
2. The profile selecting decisions currently in force.
3. A registry of concrete enforcement mechanisms.
4. Typed change contracts separating facts, inferences, assumptions, and unknowns.
5. A canonical plan and manifest generated by the private TypeScript governance host.
6. Governance runs bound to both plan and manifest that carry identity, provenance, and outcomes.

The central rule is simple: evidence may observe the law; it may never redefine it.

## Commands

```bash
npm run governance:compile  # regenerate plan.json and manifest.json
npm run governance:verify   # prove stored artifacts match deterministic compilation
npm run governance:check    # host quality checks plus verification
```

An accepted decision file is never edited. Introduce a new decision with a new ID and use
`supersedes` to replace an earlier decision. Supersession is derived in the compiled plan.

No gate may claim `enforced` unless its `boundTo` mechanism is registered and the host can verify
that mechanism exists. External mechanisms remain `deferred` until an external observer can prove
their state.

A local run can inspect the contract but cannot satisfy a gate owned by GitHub Actions. Hashes prove
integrity and freshness; trusted signatures, attestations, and repository rules prove authenticity.

Memory systems, conversations, issue comments, and generated summaries are useful context but are
not normative authority.
